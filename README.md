# Daily Quran (Chrome extension)

A Manifest V3 browser extension that opens a **side panel** with a Quranic verse (Arabic Uthmani script and English translation), optional **tafsir**, and **AI-assisted** reflection tied to the current verse. Built with React, TypeScript, Vite, and Tailwind CSS.

## Features

- **Side panel UI:** Click the toolbar icon to open the panel alongside the page you are reading (Chrome opens the side panel; behavior may differ slightly in other Chromium browsers).
- **Verse of the moment:** Loads a random verse by default; use **prev** / **next** to move through the Quran in order.
- **Tafsir:** Expand commentary for the current verse when the API provides it.
- **Ask AI:** Submit questions about the displayed verse; responses are generated by your backend (treat AI output as supplementary, not authoritative).
- **Lightweight client:** No built-in analytics in the extension code; verse and AI features depend on your configured API.

## Prerequisites

- Node.js (LTS recommended)
- A backend that implements the verse and AI endpoints expected by this client (see `src/services/verse/` and `src/services/QuestionAnsweringAi/`). The client sends `extension_secret` on requests for authentication.

## Installation

1. Clone the repository and install dependencies:

   ```bash
   npm install
   ```

2. Configure environment variables (see `.env.example`):

   ```bash
   cp .env.example .env
   ```

   Set `VITE_API_URL` to your API origin (no `/quran` suffix) and `VITE_EXTENSION_SECRET` to the secret your server expects.

3. Build the extension:

   ```bash
   npm run build
   ```

4. In **Chrome** or **Edge**, open `chrome://extensions` (or `edge://extensions`), enable **Developer mode**, click **Load unpacked**, and choose the **`dist`** folder produced by the build.

## Usage

- Pin or click the **Daily Quran** extension icon to open the **side panel**.
- Read the verse and translation; use **prev** / **next** to change verses.
- Use **Show Tafsir** or **Ask AI a question** for deeper context (requires a working API and, for AI, server support).

## Development

- `npm run dev` — Vite dev server for the UI (use together with loading the built output or your usual extension dev workflow).
- `npm run build` — Typecheck and production build to `dist/`.
- `npm run lint` — ESLint.
- `npm run prettier` / `npm run check-format` — Formatting.

Source lives under `src/`; extension assets and `manifest.json` are under `public/` and are copied into `dist/` on build.
