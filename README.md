# Daily Quran (Chrome extension)

A Manifest V3 browser extension that opens a **side panel** with a Quranic verse (Arabic Uthmani script and English translation), optional **tafsir**, **AI-assisted** reflection, and **Quran Foundation account integration** for saving favorite ayahs. Built with React, TypeScript, Vite, and Tailwind CSS.

## Features

- **Side panel UI:** Click the toolbar icon to open the panel alongside the page you are reading.
- **Verse of the moment:** Loads a random verse by default; use **prev** / **next** to move through the Quran in order.
- **Tafsir:** Expand commentary for the current verse when the API provides it.
- **Ask AI:** Submit questions about the displayed verse; responses are generated by your backend (treat AI output as supplementary, not authoritative).
- **Guest mode:** Random verse, prev/next, tafsir, and AI all work without any login.
- **Login with Quran.com:** Authenticate via your Quran Foundation account using OAuth2. The backend handles the OAuth flow; the extension only opens the login tab and polls for the session.
- **Save favorites (heart button):** While logged in, tap the heart on any verse to add it to your default Quran.com bookmark collection. Guests see a prompt to log in.
- **View Favorites:** When logged in, a footer button opens `quran.com/profile` in a new tab.

## Prerequisites

- Node.js (LTS recommended)
- The companion NestJS backend (`quran-chrome-extension-api`) deployed and reachable. That backend must have the QF OAuth2 credentials and Upstash Redis configured (see the backend README for details).

## Environment variables

Copy `.env.example` to `.env` and fill in the two values:

```
VITE_API_URL=https://your-api.vercel.app   # backend origin, no trailing slash
VITE_EXTENSION_SECRET=your-shared-secret
```

No QF OAuth credentials live in the extension — the backend owns them.

## Installation

1. Clone the repository and install dependencies:

   ```bash
   npm install
   ```

2. Configure environment variables:

   ```bash
   cp .env.example .env
   # edit VITE_API_URL and VITE_EXTENSION_SECRET
   ```

3. Build the extension:

   ```bash
   npm run build
   ```

4. In **Chrome** or **Edge**, open `chrome://extensions` (or `edge://extensions`), enable **Developer mode**, click **Load unpacked**, and choose the **`dist`** folder produced by the build.

## Login flow

1. Click **Login with Quran.com** in the header.
2. The extension opens a new tab pointing to `GET ${VITE_API_URL}/auth/quran/login?state=<uuid>`.
3. The backend redirects to the Quran Foundation OAuth2 consent page (PKCE).
4. After the user authorizes, the backend stores the session and renders a self-closing HTML page.
5. The extension polls `GET /auth/quran/session/<uuid>` every 1.5 s (up to 2 min). On success it stores the session token in `chrome.storage.local` and closes the login tab automatically.
6. All subsequent API requests include `x-session-token` automatically via an axios interceptor.

## Usage

- Pin or click the **Daily Quran** extension icon to open the **side panel**.
- Use **prev** / **next** to navigate; guest features work without any login.
- Log in to save verses with the heart button; the filled heart persists across sessions.
- **View Favorites** (footer, logged-in only) opens your Quran.com profile.
- **Logout** clears the local session; the extension returns to guest mode.

## Development

- `npm run dev` — Vite dev server for the UI.
- `npm run build` — Typecheck and production build to `dist/`.
- `npm run lint` — ESLint.
- `npm run prettier` / `npm run check-format` — Formatting.

Source lives under `src/`; extension assets and `manifest.json` are under `public/` and are copied into `dist/` on build.
